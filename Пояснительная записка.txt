Работу с событиями "Оплата" и "Продажа" мы реализовали через систему статусов Заказа.
Управление статусами производится через форму, вызываемую кнопкой "Статус" на форме со списком Заказов:

При создании Заказ автоматически получает статус "Черновик". Заказ в этом статусе может быть удален (например, как ошибочно заведенный). Заказы с дюбыми иными статусами остаются в системе.
Из статуса "Черновик" Заказ можно перевести в статус "Бронь", что закрепляет его за Клиентом и позволяет впоследствии произвести оплату либо отменить Заказ с указанием причины отмены (для аналитики).
Из статуса "Бронь" Заказ может быть переведён либо в статус "Оплата", либо в статус "Отмена".
Перевод Заказа из статуса "Бронь" в статус "Оплата" приводит к генерации документа "Оплата" согласно ТЗ.
Перевод Заказа из статуса "Бронь" в статус "Отмена" производится с обязательным указанием причины отмены (причина выбирается из справочника, справочник пока не редактируется). Причина может быть впоследствии отредактирована.
Из статуса "Оплата" Заказ может быть переведн в статус "Продажа". При этом генерируется документ "Продажа" (согласно ТЗ) с фиксацией наличия или отсутствия брони Отеля. Факт фиксации брони Отеля может быть впоследствии отредактирован

Каждый раз при смене статуса фиксируется дата смены статуса. Текущий статус документа отображается в таблице заказов (с указанием причины в случае статуса "Отмена").

Дополнительные улучшения

Мы решили, что со временем некоторых сущностей может стать так много, что искать их глазами в таблице станет неудобно, поэтому реализовали поиск по основным значимым полям в некоторых формах со списками:
1. В форме "Список Контактов" поиск возможен либо по части фамилии, либо по части имени, либо по части адреса электронной почты (но не все разом). Кнопка "Поиск" срабатывает при нажатии ENTER.
2. В форме "Список Клиентов" фильтрация осуществляется по его названию Клиента либо по части имени его Контактного лица.
3. В форме "Список отелей" поиск возможен по их названию и местонахождению.
4. В Форме "Список заказов" поиск заказа осуществляется по имени клиента, виду оплаты и статусу Заказа.
